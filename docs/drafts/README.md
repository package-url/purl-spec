This README explains the use of the `docs/drafts` documentation folder for 
the **purl-spec** project.

Most changes to project documentation should be handled with issues and pull
requests, but there are use cases where it may make sense to separate
new documentation files that merit significant community discussion before 
publishing them as "official" project documentation:
 - Major new documentation files
 - Major rewrite of an existing file
 - Experimental documentation (format or content)

The reason to create a new documentation file in the `docs/drafts` folder is 
to make review and comments easier to follow than if the draft document were 
kept in PR format.

If the document is a rewrite of an existing document the suggested naming 
convention is use the add "draft-v0.n" to the name.
