# purl-spec workflow files

This file summarizes the names, roles and functions of the purl-spec workflows, including but not limited to the process of submitting a PR proposing a new PURL `type`.  The majority of the files below are located in [etc/scripts/](https://github.com/package-url/purl-spec/blob/main/etc/scripts/).

| File                                                                                                                                                                                                                                                  | Description                                                                                                                                                                                                                                                                                                                                                                              |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Makefile](../../Makefile) | Top-level script. Commands include `conf`, `check`, `gendocs` (which runs `etc/scripts/generate_index_and_docs.py`) and `gencode`. |
| [CONTRIBUTING.md](../../CONTRIBUTING.md) | Among other topics, this file covers documentation, pull request guidelines, and development setup and usage instructions. |
| [.github/workflows/generate-index-and-docs.yml](../../.github/workflows/generate-index-and-docs.yml) | Automates the running of `make conf`, `make check` and `make gendocs`.  Triggered automatically by a push to `main` changing any `.json` file in `types/` or `schemas/` (or any of their respective subfolders).  Also includes a manual trigger.  Both the automatic and manual triggers open a PR that needs to be reviewed, approved and merged into `main`. |
| [etc/scripts/archive/git-split-file.sh](../../etc/scripts/archive/git-split-file.sh) | One-time use for initial mega-PR to create schema and test files. Archived. |
| [etc/scripts/format_json.py](../../etc/scripts/format_json.py) | Basic formatting (e.g., indents, line endings). Called by the Makefile `formatjson` command (which is also called by the `format` command). Not used by any other purl-spec files. Targets `schemas/`, `types/` and `tests/`. |
| [etc/scripts/generate_index_and_docs.py](../../etc/scripts/generate_index_and_docs.py) | Called by the Makefile `gendocs` command (which is also run by `.github/workflows/generate-index-and-docs.yml`). Not used by any other purl-spec files. Generates `/types/*-definition.json`, `/docs/types/definitions/*-definition.md` and `/purl-types-index.json` files. |
| [etc/scripts/purl_test.py](../../etc/scripts/purl_test.py) | Generated by Makefile `gencode` command. Not called by any file in the repo. Uses `datamodel-codegen` (requirements.txt: `datamodel-code-generator==0.31.2`) to convert JSON schemas to Pydantic models (Python classes that represent the exact same structure as the schema). Used for `.json` validation (e.g., `types/npm-definition.json`) and implementing PURL validators. |
| [etc/scripts/purl_type_definition.py](../../etc/scripts/purl_type_definition.py) | Generated by Makefile `gencode` command. |
| [etc/scripts/purl_types_index.py](../../etc/scripts/purl_types_index.py) | Generated by Makefile `gencode` command. |
| [etc/scripts/pyproject.toml](../../etc/scripts/pyproject.toml) | Configuration for Python ruff tool (linter and code formatter). Appears to use ruff for basic formatting (linting, isort etc.). |
| [etc/scripts/remove_duplicate_tests.py](../../etc/scripts/remove_duplicate_tests.py) | Identifies duplicate test cases (i.e., 100% key-value match between test objects in a single `*-test.json` file, regardless of sort order). The script round-trips each file with `json.loads()` and `json.dumps()`, which might coincidentally correct some errors such as removing blank lines between adjacent items, but the process is not looking for, identifying or fixing syntactic or other errors in the test objects. We might want to incorporate a variation of this process into the `generate-index-and-docs.yml` workflow to check test file content. |
| [etc/scripts/requirements.txt](../../etc/scripts/requirements.txt) | Python dependency manifest - includes packageurl-python. |
